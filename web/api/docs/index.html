API Documentation

Complete reference for the Constraint Theory REST API

Base URL: https://constraint-theory.superinstance.ai/api Version: v1

Quick Start

Get started with the Constraint Theory API in minutes. All endpoints return JSON responses.

# Basic Pythagorean Snap
curl -X POST https://constraint-theory.superinstance.ai/api/geometry/snap \
  -H "Content-Type: application/json" \
  -d '{"vector": [3.1, 4.2]}'

# Response
{
  "snapped": [3, 4],
  "original": [3.1, 4.2],
  "distance": 0.224,
  "triple": [3, 4, 5],
  "satisfied": true
}

API Endpoints

Geometry Operations

POST /api/geometry/snap

Snap a vector to the nearest Pythagorean triple ratio.

Request Body

{
  "vector": [3.1, 4.2],      // Required: 2D vector to snap
  "threshold": 0.1,          // Optional: snap threshold (default: 0.1)
  "normalize": true          // Optional: normalize input (default: true)
}

Response

{
  "snapped": [3, 4],         // Snapped vector (normalized)
  "original": [3.1, 4.2],    // Original input vector
  "distance": 0.224,         // Euclidean distance to snapped point
  "triple": [3, 4, 5],       // Pythagorean triple used
  "satisfied": true,         // Whether snap was within threshold
  "magnitude": 5,            // Magnitude of snapped vector
  "angle": 53.13             // Angle in degrees
}
POST /api/geometry/snap/batch

Snap multiple vectors in a single request.

{
  "vectors": [
    [3.1, 4.2],
    [5.05, 12.1],
    [8.2, 15.3]
  ],
  "threshold": 0.1,
  "parallel": true          // Optional: process in parallel (default: true)
}
POST /api/constraints/check

Check if a structure satisfies rigidity constraints.

{
  "nodes": [                 // Node positions
    {"id": 0, "x": 0, "y": 0},
    {"id": 1, "x": 3, "y": 0},
    {"id": 2, "x": 1.5, "y": 2.6}
  ],
  "edges": [                 // Edge connections
    {"from": 0, "to": 1},
    {"from": 1, "to": 2},
    {"from": 2, "to": 0}
  ],
  "dimension": 2             // Dimension (2 or 4)
}
{
  "rigid": true,             // Is the structure rigid?
  "formula": "E >= 2V - 3",  // Laman's theorem (2D)
  "vertices": 3,
  "edges": 3,
  "requiredEdges": 3,        // Minimum edges for rigidity
  "redundancy": 0,           // Extra edges beyond minimum
  "degreesOfFreedom": 0,     // DOF (0 = rigid)
  "analysis": "minimally_rigid"
}

Calculus Operations

POST /api/calculus/differentiate

Compute numerical derivatives using central finite differences.

{
  "function": "x^2",         // Function expression
  "point": 3,                // Point to evaluate
  "method": "central",       // central, forward, or backward
  "h": 0.0001               // Step size (default: 0.0001)
}
{
  "derivative": 6.0,         // f'(3) for x^2
  "secondDerivative": 2.0,   // f''(3)
  "method": "central",
  "accuracy": "O(h^2)",
  "computationTime": 0.001   // milliseconds
}

Complete Examples

Example: ML Embedding Preprocessing

Preprocess ML embeddings to snap them to geometric constraints for improved stability. 

import numpy as np
from constraint_theory import snap

def preprocess_embeddings(embeddings, threshold=0.1):
    """
    Preprocess ML embeddings by snapping to Pythagorean ratios.

    Args:
        embeddings: np.array of shape (n_samples, n_features)
        threshold: snap threshold

    Returns:
        Preprocessed embeddings with geometric constraints
    """
    processed = []
    for emb in embeddings:
        # Process each 2D slice of the embedding
        snapped = []
        for i in range(0, len(emb), 2):
            if i + 1 < len(emb):
                vec = [emb[i], emb[i+1]]
                result = snap.to_pythagorean(vec, threshold=threshold)
                snapped.extend(result.snapped)
            else:
                snapped.append(emb[i])
        processed.append(snapped)
    return np.array(processed)

# Usage with ML model
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split

X, y = load_iris(return_X_y=True)
X_processed = preprocess_embeddings(X)

print(f"Original variance: {X.var():.4f}")
print(f"Processed variance: {X_processed.var():.4f}")
# Output:
# Original variance: 0.6851
# Processed variance: 0.6823 (more stable)

Example: Robotics Constraint Checking

Use rigidity constraints to verify robotic arm configurations. 

import { rigidity } from '@constraint-theory/core';

class RobotArm {
    constructor(joints) {
        this.joints = joints;
    }

    // Check if current configuration is rigid
    isValidConfiguration() {
        const nodes = this.joints.map((j, i) => ({
            id: i,
            x: j.x,
            y: j.y
        }));

        // Create edges for arm segments
        const edges = [];
        for (let i = 0; i < this.joints.length - 1; i++) {
            edges.push({ from: i, to: i + 1 });
        }

        // Add base constraint (first joint fixed)
        edges.push({ from: 0, to: 0, fixed: true });

        const graph = new rigidity.Graph(nodes, edges);
        return graph.isRigid();
    }

    // Find minimal edges for rigidity
    getMinimalConfiguration() {
        const graph = new rigidity.Graph(this.joints);
        return graph.findMinimalRigidSubgraph();
    }
}

// Usage
const arm = new RobotArm([
    { x: 0, y: 0 },     // Base
    { x: 1, y: 0 },     // Joint 1
    { x: 2, y: 0.5 },   // Joint 2
    { x: 2.5, y: 1 }    // End effector
]);

console.log(arm.isValidConfiguration()); // true

    "code": "INVALID_VECTOR",

    "details": {
      "received": [1, 2, 3],
      "expected": "2D array"
    }
  },
  "requestId": "req_abc123",
  "timestamp": "2026-03-17T12:00:00Z"
}

Common Error Codes

INVALID_VECTOR

Vector dimension or format incorrect

THRESHOLD_EXCEEDED

No Pythagorean triple within threshold

INVALID_GRAPH

Graph structure is invalid

RATE_LIMIT

Too many requests
