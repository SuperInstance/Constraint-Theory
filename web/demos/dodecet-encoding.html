12-Bit Dodecet Encoding

Memory-Efficient Geometric Encoding

Dodecet encoding packs 3D coordinates into 12 bits using Pythagorean triples. Perfect for cellular agent positions with exact arithmetic and zero floating-point ambiguity.

Why 12 Bits?

  • 4 bits per coordinate: Each axis (X, Y, Z) gets 4 bits
  • 16 values per axis: 2⁴ = 16 discrete positions per dimension
  • 4096 total states: 16³ = 4096 unique positions in 3D space
  • Memory efficient: Only 2 bytes to store any 3D position
  • Hex-friendly: Each digit represents one coordinate (0xXYZ)

Benefits of Dodecet Encoding

💾

Memory Efficient

Store any 3D position in just 2 bytes. Compared to 3 floats (12 bytes), that's 6x compression.

🎯

Exact Arithmetic

No floating-point ambiguity. Every position is a discrete, exact value.

⚡

Fast Comparison

Single integer comparison instead of three float comparisons.

🔢

Hex-Friendly

Easy to read: 0x123 = (1, 2, 3). Perfect for debugging and logging.
