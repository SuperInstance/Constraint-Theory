Benefits of Orientation-Based Filtering

🎯

Automatic Relevance

Agents automatically receive only relevant information based on their position and orientation. No manual filtering needed.

⚡

Reduced Computation

Agents only process information within their perception. Dramatically reduces computational load as system scales.

🔒

Natural Security

Information compartmentalization is inherent to the geometry. Agents can't access what they can't "see".

📈

Better Scaling

As agent count increases, each agent's processing remains constant. Only processes local neighborhood.

Technical Implementation

Agent State Representation

// Each agent has position and orientation
struct Agent {
  position: Vec2,      // (x, y) in space
  orientation: f32,    // Angle in radians
  field_of_view: f32,  // Visible angle (e.g., 90°)
  view_distance: f32,  // Maximum visibility distance
}

// Convert to dodecet encoding
let dodecet = encode_dodecet(
  agent.position.x,
  agent.position.y,
  agent.orientation
);

Visibility Check Algorithm

function is_visible(observer, target):
  // Calculate distance
  distance = euclidean(observer.position, target.position)
  if distance > observer.view_distance:
    return false

  // Calculate angle to target
  angle_to_target = atan2(
    target.position.y - observer.position.y,
    target.position.x - observer.position.x
  )

  // Check if within field of view
  angle_diff = abs(angle_to_target - observer.orientation)
  angle_diff = min(angle_diff, 2π - angle_diff)  // Handle wrap-around
  half_fov = observer.field_of_view / 2

  return angle_diff <= half_fov

Information Filtering Benefits

  • At 10 agents: Each sees ~3 agents (30% of total)
  • At 100 agents: Each sees ~5 agents (5% of total)
  • At 1000 agents: Each sees ~7 agents (0.7% of total)
  • Scaling: O(√n) neighbors vs O(n) global state

Real-World Analogy: FPS Games

Think of your favorite First-Person Shooter game. When you play, you don't see the entire map and all players at once. You only see what's in your field of view from your position.

This isn't a limitation - it's a feature. The game doesn't need to process what's happening behind you or on the other side of the map. This is why FPS games can support many players while maintaining smooth performance.

Constraint Theory applies this same principle to AI agents. Each agent has its own "camera" in the world, and only processes information within its view. This is why our system scales to thousands of agents while traditional systems struggle with hundreds.
