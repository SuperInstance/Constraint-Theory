How KD-Tree Works

1

Build Phase

Recursively split space by median points. Alternate between X and Y dimensions at each level to create a balanced binary tree.

function buildKDTree(points, depth = 0):
  if points is empty: return null
  axis = depth % 2  # Alternate X, Y
  sort points by axis
  median = middle point
  left = buildKDTree(points before median, depth + 1)
  right = buildKDTree(points after median, depth + 1)
  return Node(median, left, right)
2

Query Phase

Traverse tree pruning branches that cannot contain neighbors. Only search relevant regions of space.

function queryNeighbors(node, target, radius, depth = 0):
  if node is null: return []
  axis = depth % 2
  distance = euclidean(node.point, target)
  neighbors = []
  if distance <= radius:
    neighbors.append(node.point)
  # Prune branches outside search space
  if target[axis] - radius <= node.point[axis]:
    neighbors += queryNeighbors(node.left, target, radius, depth + 1)
  if target[axis] + radius >= node.point[axis]:
    neighbors += queryNeighbors(node.right, target, radius, depth + 1)
  return neighbors
3

Why O(log n)?

At each level, we eliminate half the search space. With n points, tree depth is ~log₂(n). We only visit nodes that could contain neighbors.

  • Linear search: Check every point → O(n)
  • KD-tree search: Only check relevant branches → O(log n)
  • At 1000 points: ~1000 checks vs ~10 checks
  • At 10000 points: ~10000 checks vs ~13 checks
